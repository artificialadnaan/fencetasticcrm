# Fencetastic CRM — Master Sheet, Income/Expenses & Work Order Generator

**Date:** 2026-04-03
**Status:** Approved

---

## Overview

Three new subsystems for Fencetastic CRM:

1. **Master Sheet** — Spreadsheet-style project tracker with two views (grid + card), matching the existing Excel "Project Schedule" layout
2. **Income & Expenses** — Auto-populated financial ledger pulling from project data, with manual entry support
3. **Work Order Generator** — Drawing tool (React-Konva) for placing houses, drawing fence lines with specs, generating printable PDF work orders

---

## 1. Master Sheet (Project Tracker)

### Data Model Changes

Extend the existing `Project` model in Prisma:

**New fields:**
- `commissionOwed` (Decimal, optional) `@db.Decimal(10, 2)`
- `commissionPaid` (Decimal, optional) `@db.Decimal(10, 2)`
- `memesCommission` (Decimal, optional) `@db.Decimal(10, 2)`
- `aimannsCommission` (Decimal, optional) `@db.Decimal(10, 2)`

**Reused existing fields (no new columns needed):**
- `subcontractor` (String, optional) — already exists, used for the "SUB" column in grid view
- `materialsCost` (Decimal) — already exists, maps to "Materials ONLY" column
- Sub payments — derived from existing `SubcontractorPayment` relation (aggregated at query time)

**Calculated at query time (NOT stored):**
- Outstanding Receivables = `projectTotal - customerPaid`
- Outstanding Payables = `commissionOwed - commissionPaid`
- $ Profit = `moneyReceived - forecastedExpenses - commissionOwed`
- % Profit = `profitDollar / projectTotal` (guard: return 0 when `projectTotal` is 0)
- NET PROFIT $ = `profitDollar - memesCommission - aimannsCommission`
- NET Percent = `netProfitDollar / projectTotal` (guard: return 0 when `projectTotal` is 0)

**Enum changes:**
- Add `CLOSED` and `WARRANTY` to `ProjectStatus`
- Add `ZELLE` and `FINANCING` to `PaymentMethod`

**Relation additions to Project model:**
```prisma
transactions  Transaction[]
workOrders    WorkOrder[]
```

## 2. Income & Expenses

### Data Model

New `Transaction` model:

```prisma
model Transaction {
  id              String          @id @default(uuid()) @db.Uuid
  type            TransactionType
  amount          Decimal         @db.Decimal(10, 2)
  date            DateTime        @db.Timestamptz
  category        String
  description     String
  payee           String?
  paymentMethod   String?         @map("payment_method")
  projectId       String?         @map("project_id") @db.Uuid
  sourceField     String?         @map("source_field")
  isAutoGenerated Boolean         @default(false) @map("is_auto_generated")
  createdAt       DateTime        @default(now()) @map("created_at") @db.Timestamptz
  updatedAt       DateTime        @updatedAt @map("updated_at") @db.Timestamptz

  project Project? @relation(fields: [projectId], references: [id])

  @@map("transactions")
}

enum TransactionType {
  INCOME
  EXPENSE
}
```

**`sourceField`** — tracks which project field triggered the auto-generated transaction (e.g., `"customerPaid"`, `"materialsCost"`). Used for audit trail and debugging. `null` for manual entries.

**Categories (seeded, user can add more):**
- Income: Customer Payment, Deposit, Zelle, Financing
- Expense: Materials, Sub Payment, Insurance, Advertising, Accounting, Software, Tools, Gas/Travel, Misc

### Auto-Population Logic

**Implementation:** The project update service (`project.service.ts`) must be modified to:
1. Read the current project values BEFORE applying the update (`prisma.project.findUnique`)
2. Compute the delta for tracked fields: `newValue - oldValue`
3. If delta is non-zero, create a Transaction record
4. Steps 1-3 run inside a Prisma `$transaction()` block to ensure atomicity

**Tracked fields and their transaction mappings:**
| Project Field | Transaction Type | Category | sourceField |
|---|---|---|---|
| `customerPaid` | INCOME | Customer Payment | `"customerPaid"` |
| `materialsCost` | EXPENSE | Materials | `"materialsCost"` |

**SubcontractorPayment updates:** When `SubcontractorPayment.amountPaid` is updated via the existing sub payment routes, create an EXPENSE transaction with category "Sub Payment", linked to the project, `sourceField = "subPayment"`.

**Negative deltas:** If a value is corrected downward (e.g., customerPaid $10000 → $8000), a transaction with amount `-2000` is created. This preserves the full audit trail.

**Manual entries:** Added for non-project costs (rent, insurance, gas, tools) via the Add Transaction form.

**Operating Expenses:** Existing `OperatingExpense` recurring costs feed into the finances page as a separate summary section (not auto-created as transactions — they represent recurring obligations, not actual transactions).

### Auth

All transaction routes use the existing `requireAuth` middleware. No additional permission model — any authenticated user can create/edit/delete transactions.

### UI (`/finances` — new sidebar page)

**Summary bar at top:**
- Total Income (MTD / YTD toggle)
- Total Expenses (MTD / YTD toggle)
- Net Profit (MTD / YTD toggle)

**Transaction list:**
- Filterable table: type (income/expense/all), date range, category, project
- Sorted by date descending
- Income rows in green text, expense rows in red text
- Pagination
- Auto-generated entries show a small "auto" badge

**Add Transaction button:**
- Modal form: type, amount, date, category, description, payee, payment method, optional project link

**Charts:**
- Monthly income vs expenses bar chart (Recharts)
- Category breakdown pie chart

## 3. Work Order Generator

### Data Model

```prisma
model WorkOrder {
  id            String         @id @default(uuid()) @db.Uuid
  projectId     String         @map("project_id") @db.Uuid
  drawingData   Json           @map("drawing_data")
  propertyNotes String?        @map("property_notes") @db.Text
  segments      FenceSegment[]
  createdAt     DateTime       @default(now()) @map("created_at") @db.Timestamptz
  updatedAt     DateTime       @updatedAt @map("updated_at") @db.Timestamptz

  project Project @relation(fields: [projectId], references: [id])

  @@map("work_orders")
}

model FenceSegment {
  id              String     @id @default(uuid()) @db.Uuid
  workOrderId     String     @map("work_order_id") @db.Uuid
  segmentNumber   Int        @map("segment_number")
  fenceType       String     @map("fence_type")
  style           FenceStyle @default(NORMAL)
  height          Decimal    @db.Decimal(10, 2)
  linearFeet      Decimal    @map("linear_feet") @db.Decimal(10, 2)
  steps           Json?
  additions       Json       @default("[]")
  customAdditions Json       @default("[]") @map("custom_additions")
  notes           String?    @db.Text

  workOrder WorkOrder @relation(fields: [workOrderId], references: [id], onDelete: Cascade)

  @@map("fence_segments")
}

enum FenceStyle {
  NORMAL
  STEPPED
}
```

**`drawingData` JSON structure:**
- Serialized Konva stage — positions, dimensions, and types of all shapes on the canvas
- Includes house shape, property lines, fence lines, gates, labels, measurements
- **Size limit:** Validated server-side, max 5MB JSON payload. Typical drawings are well under 100KB.

**`steps` JSON structure (for stepped fencing):**
```json
[
  { "position": 0, "height": 6.0 },
  { "position": 12, "height": 5.5 },
  { "position": 24, "height": 5.0 }
]
```
- `position` = feet from start of segment
- `height` = fence height at that step

### Drawing Tool UI (`/projects/:id/work-order`)

Accessed from the project detail page via "Create Work Order" button. Only available for existing projects.

**Canvas (React-Konva):**

- Grid background with configurable scale (default: 1 square = 1 foot, adjustable)
- Zoom via scroll wheel, pan via drag on empty space
- Snap-to-grid for precise placement

**Left toolbar:**
| Tool | Behavior |
|------|----------|
| House | Drop rectangular shape, resize handles, rotate, dimension labels auto-display |
| Fence Line | Click to place points, line segments created between them. Each segment = one `FenceSegment`. Double-click to end sequence |
| Gate | Place gate marker on fence line with type label (single swing, double swing, sliding) |
| Property Line | Dashed line for property boundaries |
| Text/Label | Add annotations anywhere on canvas |
| Measurement | Click two points, shows distance in feet |
| Select | Click to select/move elements |
| Delete | Click element to remove |
| Undo/Redo | Standard undo/redo stack |

**Right panel — Segment Properties (visible when a fence segment is selected):**

- Fence type dropdown: Wood B/B, Wood S/S, Wood Raw, Iron, Chain Link, Vinyl, + custom entry
- Style toggle: Normal | Stepped
- Height input (feet)
- Linear feet (auto-calculated from drawing, editable for manual override)
- **If Stepped:**
  - List of steps with position (feet from start) and height
  - "Add Step" button to add new step markers
  - Steps visualized on the canvas as horizontal offset lines
- Additions checklist:
  - Top Cap
  - Double Trim
  - Single Trim
  - Kickboard
  - Boxed Post Covers
  - Corbels
  - Retaining Wall
  - Gate Operator
- "Add Custom" button — free text input for any custom addition
- Notes text field

**Bottom bar:**
- Total linear feet across all segments
- Segment count
- "Save" button (persists to database)
- "Generate PDF" button

### Auth

## New Routes Summary

### API

```
# Transactions (Income/Expenses)
GET    /api/transactions          — List with filters (type, date range, category, projectId)
POST   /api/transactions          — Create manual transaction
PATCH  /api/transactions/:id      — Update transaction
DELETE /api/transactions/:id      — Delete transaction
GET    /api/transactions/summary  — MTD/YTD totals for income, expenses, net

# Work Orders
GET    /api/projects/:id/work-order    — Get work order for project (includes segments)
POST   /api/projects/:id/work-order    — Create work order with segments
PATCH  /api/work-orders/:id            — Update work order (drawing + segments)
DELETE /api/work-orders/:id            — Delete work order (cascades to segments)
POST   /api/work-orders/:id/pdf        — Generate PDF (receives PNG image in body, returns PDF)

# Projects (extended)
GET    /api/projects/grid              — Grid view data (all columns, computed fields, sub payment aggregation)
PATCH  /api/projects/bulk              — Bulk update projects (status, etc.)
GET    /api/projects/export            — Excel export (server-side, returns .xlsx file)
```

## Calculated Field Formulas

Computed client-side on every render. Division-by-zero returns 0.

| Field | Formula |
|-------|---------|
| Outstanding Receivables | `projectTotal - customerPaid` |
| $ Profit | `moneyReceived - forecastedExpenses - commissionOwed` |
| % Profit | `projectTotal > 0 ? profitDollar / projectTotal : 0` |
| Meme's Comm | `projectTotal * 0.05` (or manual override) |
| Aimann's Comm | `profitDollar * 0.25` (or manual override) |
| NET PROFIT $ | `profitDollar - memesCommission - aimannsCommission` |
| NET Percent | `projectTotal > 0 ? netProfitDollar / projectTotal : 0` |
| Outstanding Payables | `commissionOwed - commissionPaid` |
